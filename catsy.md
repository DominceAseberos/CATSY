# 1\. Introduction

CUTshier is a Point of Sale and Inventory Management System developed for Cutsy Coffee, a neighborhood café located at Purok A, Suaybaguio Street, Barangay Magugpo North, Tagum City. The system aims to replace the café's current manual processes for order taking, sales tracking, inventory management, and customer reservations with a modern, automated solution.

## 1.1 System Name

CUTshier: A POS and Inventory Management System for Cutsy Coffee

## 1.2 Purpose

This document defines the complete functional and non-functional requirements for CUTshier, incorporating all refinements and decisions made during the requirements gathering process.

## 1.3 Scope

The system covers the following:

- Automation of sales transactions and order processing
- Real-time daily and monthly sales reports
- QR-based stamp card loyalty system
- Inventory tracking with automatic stock updates
- Product sales analysis for best-selling and slow-moving items
- Online table reservation for customer convenience
- Offline-first mobile POS app with auto-sync
- Role-based access for admin, staff, and customers

The system does not cover:

- Financial accounting and payroll management
- Third-party delivery or courier service integration

# 2\. System Architecture

## 2.1 Architectural Pattern

CUTshier follows a Client-Server architecture with MVC (Model-View-Controller) applied on the server side for web interfaces, and a REST API for communication with the mobile POS application. A separate Python ML microservice handles all machine learning logic and is called by the main backend API when the admin views reports.

## 2.2 Platforms & Access

| **Platform**           | **User**        | **Access**                                                                    |
| ---------------------- | --------------- | ----------------------------------------------------------------------------- |
| Customer web portal    | Customer        | Browse menu, reserve table, view QR, view stamps, purchase history            |
| Mobile POS app (APK)   | Staff / Cashier | Process orders, manage seats, scan QR, handle payments and refunds            |
| Admin web panel        | Admin / Owner   | Reports, analytics, inventory, CMS, employee management, APK download         |
| ML microservice (internal) | Backend API | Sales forecasting, product classification, restock prediction (admin-only)    |

## 2.3 Offline-First Mobile Strategy

The mobile POS app uses SQLite as a local data store, enabling core POS operations without internet connectivity. Auto-sync triggers immediately when internet is restored. Server data always takes priority in conflict resolution.

- Works offline: order processing, inventory viewing, order modification, receipt issuance
- Requires internet: QR stamp crediting, reservations, live seat sync, admin panel data
- QR scans while offline are queued and synced on reconnect, with staff notified of pending status

# 5\. Loyalty Stamp System Summary

The following summarizes the complete stamp system logic for reference:

- 1 stamp is awarded per eligible product in a completed and paid transaction, linked to a logged-in customer account.
- Only products marked as eligible by the admin count toward stamps.
- The stamp card displays a 3x3 visual grid of 9 slots (X = filled, O = empty).
- The claim button activates only once all 9 slots are filled.
- If new stamps exceed 9, the grid fills to 9 and excess stamps are queued as a separate count for the next cycle.
- Upon claiming, the system deducts 9 stamps and resets the card to 0, then fills the excess stamps into the new cycle (e.g., 12 total stamps - claim - resets to 3).
- Staff links the customer account after payment via QR code scan or member list selection on the handheld device.
- Reward claiming via web: customer picks a reward from a list, generates a unique QR or text code, and presents it to staff in-store.
- Each reward code is single-use only and expires once redeemed by staff.
- If a stamp scan occurs offline, it is queued and synced when internet is restored. Staff are warned of the pending status.
- Stamps earned from a refunded order are deducted from the customer stamp count upon successful refund.
- Admin configures which products are stamp-eligible and manages the list of claimable reward items.
